Evaluasi Divisi IT KPU KMTETI

1. Job Desk

  • Membuat rancangan kasar website KPU KMTETI
  • Membuat website KPU KMTETI
  • Membuat alur tata cara pemilihan di website KPU KMTETI
  • Menjaga keamanan website selama berlangsungnya pemilihan
  • Mengurus channel YouTube KPU KMTETI

Jobdesk Subdivisi

  • UI/UX: Bikin rancangan website KPU KMTETI di Figma berdasarkan grand design
  • Frontend: Mengaplikasikan rancangan ke dalam kodingan website menggunakan Next.js dan Tailwind
  • Backend: Membuat sistematika voting menggunakan Firebase API

2. Kendala

  • Tidak jelas fix rilis website kapan
  • Grand design dari DMD ga rapi sehingga UI/UX kesusahan
  • Kurang komunikatif, inisiatif, dan partisipasi (mostly di UI/UX)
  • Akun YouTube KPU KMTETI belum disetting live
  • Live orasi ga bisa diunduh
  • Membernya pada ikut TETI Programming Week sehingga off 1 hari penuh
  • Display tata cara di iPhone
  • Firebasenya kena serang

3. Penyelesaian

  • Akhirnya ini H-2 pemilihan baru rilis website
  • Dibuatkan grup untuk DMD dan IT agar bisa musyawarah
  • Member subdivisi lain seperti BE ikut membantu FE dan FE ikut membantu UI/UX
  • Memakai akun YouTube pribadi koor IT
  • Merekam orasi dengan OBS
  • Solusinya ya, kejar tayang seperti nama kami
  • Memakai iframe
  • Ada backup dari koor sehingga nilai yang tersimpan masih aman

4. Evaluasi

  • Harus selalu reach out member IT
  • Siap untuk bekerja di luar jobdesk subdivisi
  • Tanyakan tenggat website kapan (kasih tenggat agar membernya mau gerak)
  • Jangan lupa rules Firebase-nya disetting juga biar ga kena serang
  • Tambahin Boolean di Firestore Database untuk track suara calon1, calon2, dst. jikalau realtime databasenya error
  • Yang ngurusin live YouTube ternyata IT, yang 2024 malah DMD yang ngurusin
  • Bagi job, jangan paksain semuanya ke 1 orang. Kalau 1 orang itu sibuk, malah engga jalan pengerjaannya
